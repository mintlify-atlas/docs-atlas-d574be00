## Install and use a skill

This guide walks you through installing and using the `github-write-pr-message` skill, which generates well-structured PR messages from your git changes.

<Steps>
  <Step title="Add the marketplace">
    First, add the Skills Marketplace to Claude Code:
    
    ```shell
    /plugin marketplace add fuongz/skills
    ```
  </Step>
  
  <Step title="Install the skill">
    Install the `github-write-pr-message` skill:
    
    ```shell
    /plugin install github-write-pr-message@fuongz-skills
    ```
    
    The skill is now available for use in your Claude Code session.
  </Step>
  
  <Step title="Make some changes">
    Create a branch and make some code changes in your git repository:
    
    ```shell
    git checkout -b feat/add-user-settings
    # Make your changes
    git add .
    git commit -m "Add user settings page with theme toggle"
    ```
  </Step>
  
  <Step title="Generate a PR message">
    Run the skill to generate a PR message:
    
    ```shell
    /github-write-pr-message
    ```
    
    Or provide additional context:
    
    ```shell
    /github-write-pr-message focus on the theme toggle feature
    ```
  </Step>
  
  <Step title="Review the output">
    The skill will analyze your changes and generate a complete PR message including:
    
    - A descriptive PR title following conventional commit format
    - Summary of what changed and why
    - List of specific changes
    - Related issue numbers (extracted from branch name or commits)
    - Type of change checklist
    - Testing checklist
    - General PR checklist
    
    The output is ready to paste into GitHub or use with `gh pr create`.
  </Step>
</Steps>

## How it works

The `github-write-pr-message` skill intelligently analyzes your repository:

1. **Reads git context:** Branch name, commits, changed files, and diff
2. **Detects PR templates:** Checks for existing templates in `.github/`, `docs/`, or repo root
3. **Analyzes changes:** Understands what changed, why it changed, and the impact
4. **Generates output:** Creates a well-structured PR title and body

## Template detection

The skill automatically detects and fills in PR templates from these locations (in priority order):

1. `.github/pull_request_template.md`
2. `.github/PULL_REQUEST_TEMPLATE/*.md`
3. `docs/pull_request_template.md`
4. `pull_request_template.md`

If no template is found, it uses a comprehensive default structure.

## Example output

Here's what a generated PR message looks like:

<CodeGroup>
```markdown Title
feat: add user settings page with theme toggle
```

```markdown Body
## Summary

Implements a new user settings page that allows users to customize their experience. The main feature is a theme toggle that switches between light and dark modes.

## Changes

- Added new UserSettings component with theme toggle
- Implemented theme context provider
- Updated navigation to include settings link
- Added user preferences storage using localStorage

## Related Issues

Closes #42

## Type of Change

- [x] New feature
- [ ] Bug fix
- [ ] Breaking change
- [ ] Refactoring
- [ ] Documentation update

## Testing

- [x] Unit tests pass
- [x] Integration tests pass
- [x] Manually tested

## Checklist

- [x] Self-review completed
- [x] Code follows project style guidelines
- [x] No new warnings introduced
- [x] Tests added where applicable
```
</CodeGroup>

<Note>
  The skill follows GitHub's closing keywords (`Closes`, `Fixes`, `Resolves`) to automatically link and close issues when the PR is merged.
</Note>

## Advanced usage

You can provide additional context to focus the PR message:

```shell
/github-write-pr-message focus on security improvements
/github-write-pr-message emphasize the performance optimizations
/github-write-pr-message highlight breaking changes in the API
```

The skill will incorporate this context when analyzing your changes and generating the message.
